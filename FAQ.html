About

A. 1.0    What is FAQtor?


FAQtor is short for "Frequently Asked Questions generaTOR".

Return to top  

A. 1.1    What does FAQtor do?


FAQtor is a python script that enables you to easily create and maintain a list of questions and answers in a configurable format.

Return to top  

A. 1.2    Can you show me a webpage that uses FAQtor?


The page you are viewing was generated by FAQtor

Return to top  

A. 1.3    Who developed FAQtor?


Phil Schwartz

Dan Nygren made a few additions to Phil's excellent work to allow pointing a reader to a particular section of the FAQ (e.g. http://www.example.com/FAQ.html#About ), set the date last updated automatically, support for Python 2.7, and then Python 3.X .

Return to top  

A. 1.4    Why didn't you feed back these changes to the original FAQtor?


I did make an attempt, but was unable to successfully do so. Hosting my changes per the license on github seemed to be the right thing to do.

Usage

A. 2.0    What do I need in order to use FAQtor?


Python 3 is required. FAQtor was tested with Python 3.5.3

Return to top  

A. 2.1    How do I run FAQtor? 


$ python faqtor.py yourfaq.xml [yourconfig.cfg]

A makefile has been provided to allow this to be done easier and keep all the FAQtor files in a FAQ subdirectory and move the generated FAQ.html to the main directory. 

$ make
faqtor.py FAQ.xml FAQ.cfg
chmod u=rw,g=r,o=r FAQ.html
cp -p FAQ.html ..

A. 2.3    What does the config file look like?


The FAQtor FAQ s also generated using the config file in the FAQtor source distribution. This file is optional. If not supplied, defaults are provided. See the FAQ.cfg file in the FAQtor source distribution.

Although FAQtor will use defaults, it is recommended that you create a config file to customize the layout of the FAQ.

A. 3.1    How do I refer to answers (from other documents or from within the FAQ itself)?


In the XML input file, within each answer section that you wish to refer to you must assign a name-value entity. Only answers that you wish to refer to require this additional syntax, which looks like this: 
Referring to Answers
http://www.example.com/FAQ.html#referring to answers
file:///home/dnygren/FAQtor/FAQ.html#referring to answers

Now, anywhere you wish to refer to it, you simply use the anchor with the name you just created. This name can be anything as long as it adheres to HTML syntax rules.

You can use the URL in the summary of questions at the top of the FAQ, however this can result in pointing to the wrong question if you don't add questions sequentially and then get the question numbers out of order. 
Referring to Answers
http://www.example.com/FAQ.html#3_1
file:///home/dnygren/SW/Git/FAQtor/FAQ.html#3_1
